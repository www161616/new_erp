# PRD — 條碼模組（Barcode Module）

> 零售連鎖 ERP，總倉 1 + 門市 100 + SKU 15,000。
> 本模組為**橫向基礎模組（Cross-cutting）**，被採購、配貨、銷售、庫存盤點共用。
> v0.1 checklist 版。

---

## 1. 模組定位
- [ ] 提供「條碼 ↔ 商品 / 單據 / 批號 / 箱號」的對應與掃碼服務
- [ ] **不**擁有商品、庫存、單據的業務邏輯；只負責「讓掃描器一掃就找到正確的東西」
- [ ] 是採購收貨、門市收貨、銷售、盤點的共同入口
- [ ] 同時涵蓋：**條碼識別（Scan In）** + **條碼產生 / 列印（Generate Out）**

---

## 2. 使用情境（Scan In）

| 情境 | 誰 | 掃什麼 | 系統反應 |
|---|---|---|---|
| 總倉收貨 | 倉管 | 商品條碼 | 掛到當前 PO，數量 +1 |
| 總倉配貨 | 倉管 | 商品條碼 / 箱條碼 | 掛到當前配貨單，出庫 |
| 門市收貨 | 店長 | 配貨單條碼 / 商品條碼 | 確認收貨、庫存入帳 |
| 門市銷貨（POS）| 店員 | 商品條碼 | 加入購物車、扣庫存 |
| 盤點 | 倉管 / 店員 | 商品條碼 | 累加到盤點單實盤數 |
| 查詢 | 任一角色 | 商品條碼 | 顯示庫存 / 價格 / 規格 |
| 退貨 | 店員 / 倉管 | 商品條碼（含原發票條碼）| 帶入退貨明細 |

---

## 3. 條碼類型

- [ ] **商品條碼（Product Barcode）**
  - [ ] 原廠條碼：EAN-13、EAN-8、UPC-A、UPC-E、CODE-128
  - [ ] 內部條碼：系統自產（例：`LT` + 日期 + 流水碼 + Check digit），給無原廠碼或自製商品用
  - [ ] 同一 SKU 可有**多條碼**（原廠 + 內部 + 替換包裝），必須指定一個 `primary`
- [ ] **單據條碼（Document Barcode）**
  - [ ] PO 單號、配貨單號、收貨單號、盤點單號各自可生成條碼
  - [ ] 用於「掃單號快速打開單據」
- [ ] **箱 / 棧板條碼（P1）**
  - [ ] SSCC / ITF-14 — 一箱多件情境用
- [ ] **批號條碼（P1，視業態）**
  - [ ] GS1-128 含 AI (10) 批號、(17) 效期
- [ ] **序號條碼（P2）**
  - [ ] 單一商品唯一識別（3C、高單價）

---

## 4. Goals
- [ ] G1 — 門市 POS 銷售掃碼時間 ≤ 300ms（端到端，含庫存扣減）
- [ ] G2 — 商品條碼識別率 ≥ 99%（剩餘 1% 走「未知條碼」流程）
- [ ] G3 — 無原廠條碼商品可在 10 秒內產生內部條碼並列印標籤
- [ ] G4 — 支援多條碼 / 別名碼，打破「一個 SKU 只能綁一個條碼」的舊限制
- [ ] G5 — 未知條碼第一次掃到 → 當場可綁定 SKU；下次全集團自動認得

---

## 5. Non-Goals（v1 不做）
- [ ] ❌ **RFID / NFC**
- [ ] ❌ **二維影像 OCR 辨識商品外觀（無條碼時拍照辨識）** — P2
- [ ] ❌ **GS1 DataBar 複雜編碼**（有需要再開）
- [ ] ❌ **電子發票條碼整合** — 交由銷售 / 發票模組
- [ ] ❌ **自建條碼列印機驅動** — 沿用 BarTender 或現有標籤機

---

## 6. User Stories

### 倉管 / 店員（掃碼端）
- [ ] 作為倉管，我要能拿 USB 掃描槍對著商品一掃，系統自動帶入當前單據的一列明細，數量 +1
- [ ] 作為店員，我要能在 POS 掃 10 件不同商品連續結帳，速度不卡頓
- [ ] 作為倉管，我掃到一個系統不認得的條碼時，要能**當場綁定**到某個 SKU，下次就認得
- [ ] 作為倉管，我要能用手機相機掃（現場沒帶槍時）

### 主檔管理 / 小幫手（產生端）
- [ ] 作為主檔管理員，建立新商品時要能**一鍵產生內部條碼**（若無原廠碼）
- [ ] 作為主檔管理員，我要能為同一個 SKU 追加「替換條碼 / 別名碼」
- [ ] 作為主檔管理員，我要能**批次列印標籤**（指定 SKU 清單 + 份數）
- [ ] 作為小幫手，我收到新進的無條碼商品時，要能立即產生條碼 + 列印貼上

### 店長
- [ ] 作為店長，我要能掃「配貨單條碼」直接打開該張單據進行收貨
- [ ] 作為店長，我要能查某商品條碼 → 本店庫存、售價、近期銷量

## 7. Functional Requirements

### 7.1 條碼資料模型
- [ ] `barcodes` 表主鍵為 `barcode_value`（+ tenant_id），unique
- [ ] 欄位：`barcode_value, tenant_id, sku_id, type(ean/upc/code128/internal), is_primary, unit(個/盒/箱), pack_qty, created_at, created_by, status(active/retired)`
- [ ] 查詢 API 吞吐：單次掃碼 < 50ms（含 DB hit），使用索引 + 熱資料快取

### 7.2 條碼識別 API
- [ ] `GET /barcode/lookup?value=xxx&context=pos|gr|transfer|stocktake` → 回傳 SKU / 商品 / 單位 / 包裝數
- [ ] 優先順序：商品條碼 → 單據條碼 → 未知（回傳建議行動）
- [ ] 多條碼同值（應不可能因 unique，但需 defensive）→ 回錯誤 log
- [ ] **Context aware**：同一條碼在 POS vs 在收貨，可能需要不同處理（POS 走銷售價、GR 帶成本）

### 7.3 未知條碼處理
- [ ] 掃到未註冊條碼 → 回應「未知」+ 引導：
  - [ ] 選項 A：綁定到既有 SKU（最常見）
  - [ ] 選項 B：建立新商品主檔
  - [ ] 選項 C：標記忽略（一次性誤掃）
- [ ] 權限足夠才能當場綁定；否則存入「待處理條碼清單」，由主檔管理員處理

### 7.4 條碼產生
- [ ] **內部條碼規則（建議）**：`LT` + `yyMMdd` + 5 碼流水 + 1 碼 check digit（CODE-128，共 14 碼）
- [ ] 不重複保證：序號池集中產生
- [ ] 可批次產生 N 組預留碼，供印標機預先產碼

### 7.5 標籤列印
- [ ] 支援場景：
  - [ ] **商品價格標**（條碼 + 品名 + 售價）
  - [ ] **貨架標**（條碼 + 品名 + 儲位 + 單位）
  - [ ] **箱標**（箱條碼 + 內含品項）— P1
  - [ ] **效期標**（條碼 + 批號 + 效期）— P1
- [ ] 整合方式：
  - [ ] v1：輸出 CSV / PDF，餵給 **BarTender**（使用者既有工具）
  - [ ] P1：直接呼叫標籤機 API（需確認機型）
- [ ] 批次列印：選 SKU 清單 → 指定份數 / 模板 → 產生列印檔

### 7.6 多條碼 / 別名
- [ ] 同一 SKU 允許多筆 `barcodes` 紀錄
- [ ] `is_primary` 決定列印時用哪一個
- [ ] 舊條碼壞掉退役：`status = retired`，保留歷史對應（掃到舊條碼仍可識別、提示已退役）

### 7.7 掃描輸入裝置抽象
- [ ] **USB 掃描槍**：當鍵盤輸入處理（Enter 結尾），需要 focus 正確的輸入框
- [ ] **藍牙掃描槍**：同上
- [ ] **手機 Camera**：Web 端 `@zxing/library` 或 PWA + `BarcodeDetector API`；行動 APP 用 ML Kit / native
- [ ] **PDA**：Android 專用 APP，intent 接收條碼

### 7.8 離線容錯（POS 關鍵）
- [ ] POS 在網路斷線時：
  - [ ] 讀本地快取的 `barcode → sku` 對應表（每日同步）
  - [ ] 銷售暫存於本地，網路恢復時上拋
  - [ ] 未知條碼 → 允許暫存為「待對應」，不阻擋結帳（P1）

---

## 8. 非功能需求
- [ ] **效能**：條碼 lookup P95 < 50ms，99.9% < 200ms
- [ ] **可用性**：POS 掃碼在斷網 4 小時內仍可運作（用快取）
- [ ] **一致性**：條碼 ↔ SKU 為強一致，不可 eventually；避免 POS 掃錯東西
- [ ] **稽核**：條碼綁定 / 退役皆留操作紀錄
- [ ] **多租戶**：條碼 unique 範圍 = `tenant_id`（不同 tenant 可有相同內部碼）；原廠條碼理論上全球唯一但實務上允許 tenant 內 unique

## 10. 資料模型草稿
- [ ] `barcodes` — 條碼主檔（見 §7.1）
- [ ] `barcode_scan_logs` — 每次掃描紀錄（情境、結果、裝置、操作人；用於未來分析與問題排查；可設定保留天數）
- [ ] `internal_barcode_sequence` — 內部條碼流水號池
- [ ] `label_templates` — 標籤模板（給 BarTender / 未來列印引擎用）
- [ ] `pending_barcodes` — 未知條碼待處理佇列

---

## 11. 與其他模組整合
- [ ] **主檔模組** → 建立商品時產生條碼；維護別名條碼
- [ ] **採購模組** → 收貨頁面嵌入掃碼元件
- [ ] **庫存模組** → 盤點、調撥掃碼
- [ ] **配貨模組** → 出貨 / 收貨掃碼
- [ ] **銷售 / POS** → 結帳掃碼（本模組的**效能熱路徑**）
- [ ] **外部**：BarTender（列印）、手持掃描槍、PDA

---

## 12. 驗收準則
- [ ] 掃一個已註冊商品條碼 → 500ms 內回傳 SKU / 品名 / 單位
- [ ] 同一 SKU 綁定三個條碼，三個都能掃到同一 SKU
- [ ] 掃未知條碼 → 顯示「未知」+ 綁定 / 新增 / 忽略三按鈕
- [ ] 倉管綁定後，10 秒內第二支裝置掃同條碼已能識別
- [ ] 店員掃到退役條碼 → 仍識別為原 SKU，但 UI 提示「此條碼已退役，請改貼新條碼」
- [ ] 選 50 個 SKU 批次列印 → 產生符合 BarTender 匯入格式的檔案
- [ ] POS 斷網情境：掃常見 SKU 可正常結帳，離線資料在恢復後上拋
- [ ] 一個店員用 iPhone 相機掃條碼 → 能正確辨識 EAN-13
