## 架構速覽

```
顧客從 LINE 內點連結
  ↓
https://liff.line.me/{LIFF_ID}?store=1
  ↓ (LIFF SDK init、抓 id_token)
POST /functions/v1/liff-session (Edge Function)
  ↓ 驗 id_token + auto-register + 下載頭像到 Storage + 簽 JWT
  ↓
跳 https://new-erp-admin.vercel.app/me#token=... (Next.js app)
  ↓ 顯示會員中心（頭像、基本資料、編輯模式）
```

**兩個入口並存**：
- LIFF（LINE 內）：`https://liff.line.me/<LIFF_ID>?store=1`
- Web OAuth（外部瀏覽器）：`https://new-erp-admin.vercel.app/?store=1`

## Supabase 端當前狀態

### 已部署的 Edge Functions

| Function | 路徑 | 用途 |
|---|---|---|
| `line-oauth-start` | `/functions/v1/line-oauth-start?store=X` | 產 state、302 到 LINE authorize |
| `line-oauth-callback` | `/functions/v1/line-oauth-callback` | OAuth callback、驗 id_token、auto-register、302 回前端 |
| `liff-session` | `/functions/v1/liff-session` | LIFF SDK POST id_token → auto-register → JSON 回 JWT |
| `liff-api` | `/functions/v1/liff-api` | 會員端操作：`lookup_by_phone` / `register_and_bind` / `get_me` / `update_me` |

### Edge Function Secrets（Dashboard → Edge Functions → Secrets）

- `LINE_CHANNEL_ID` — Web OAuth 用的 channel
- `LINE_CHANNEL_SECRET`
- `LINE_CALLBACK_URL` — `https://anfyoeviuhmzzrhilwtm.supabase.co/functions/v1/line-oauth-callback`
- `LINE_STATE_SECRET` — 自己產的隨機 hex（CSRF 用）
- `LINE_LIFF_CHANNEL_ID` — LIFF 用的 channel（目前跟 LINE_CHANNEL_ID 同值）
- `MEMBER_FRONT_BASE_URL` — `https://new-erp-admin.vercel.app`
- `DEFAULT_TENANT_ID` — `00000000-0000-0000-0000-000000000001`
- `PROJECT_JWT_SECRET` — Supabase Dashboard > API > **Legacy JWT Secret**
- （Supabase 自動注入：`SUPABASE_URL` / `SUPABASE_SERVICE_ROLE_KEY`）

## 已知小坑

1. **Vercel project 名字叫 `new-erp-admin` 但實際部署的是 member app** — 當初設錯 project name，網址 `new-erp-admin.vercel.app` 其實是 member。功能沒影響但會造成混淆。想改名去 Vercel Settings → General → Project Name。

2. **Supabase JWT 切到 ECC P-256**、PostgREST 不接受我們自簽的 HS256 token。所以會員端 DB 操作全走 `liff-api` Edge Function（service_role 繞過驗證），不走 PostgREST。之後如果 Supabase 強制淘汰 Legacy HS256、我們的 `PROJECT_JWT_SECRET` 可能失效，到時要遷移到：改用 Supabase Auth admin API 建 user + 用官方發的 token。

3. **root `package-lock.json` 跟 `apps/member` workspace 還沒同步** — CI 用 `npm install`（非 `npm ci`）權宜。某天在有 Node 的電腦跑一次 `npm install` 提交 lock file 後可改回。

4. **admin 有 LIFF 註冊流程產出的會員但 admin 界面沒有「LINE 註冊來源」標籤** — 想區分 LIFF-registered vs 店員手動建立的，要加一個 `source` 欄位或看 `line_user_id IS NOT NULL`。
