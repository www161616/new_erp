# 系統立場 — C 混合型

> **這是 new_erp 所有模組的決策基準**。
> 未來任何 PRD / schema / RPC / RLS 設計有疑義時，先讀這份、再做決定。

---

## 裁定

**立場 = C 混合型：加盟店營運自主 + 總部數據可視 + 少數項目總部統一。**

**核心原則：看得到 ≠ 管得到。**

---

## 背景 — 為什麼要裁這個

之前做通知、訂單、會員、應付帳款四個模組時，每個模組自己決定了「誰有決定權、誰看得到什麼」，但**從未統一表態**。使用者於 2026-04-23 在 AP PRD Q2 預算管控時問：「這套系統是以什麼為出發點？不同角度決策會不一樣。」於是把這個隱含立場明文化。

### 使用者回答（確認方向）
- 總部收入來源：**兩者都有**（加盟金 / 權利金 / 訓練費 **+** 總倉批貨差價）
- 想看每家加盟店的營業額 / 利潤 / 客單價：**Yes**
- 推論出的立場：**C 混合型**（看得到但不管）

---

## 三層劃分

| 層面 | 立場 | 範例 |
|---|---|---|
| **數據可視性** | 總部**看得到全部** | 每家營業額 / 進貨量 / 客單 / 費用；算權利金抽成、估總倉差價、識別爛店 |
| **日常營運** | 加盟店**自主** | 發票、LINE OA、促銷、人事、雜支、預算、付款方式 |
| **總部統一項** | 總部決定、加盟店遵守 | 供應鏈（總倉）、品牌、會員系統、ERP 系統本身、發票廠商選定 |

---

## 已套用此立場的 PRD 決策（證據）

### 通知模組 v0.2（`PRD-通知模組.md` Q12）
- 每家加盟店**自主**決定 LINE OA 模式（`full` / `simple` / `none`）
- 總部**不負擔**加盟店通知成本

### 訂單取貨模組 v0.1.1（`PRD-訂單取貨模組.md` §7）
- 付款方式 `stores.allowed_payment_methods` **per-store 設定**
- 員工折扣率 `stores.employee_discount_rate` **per-store**

### 會員模組（`PRD-會員模組.md`）
- 會員綁定 `(member_id, store_id)` **多對多**（加盟店各自管顧客）

### 應付帳款模組 v0.2（`PRD-應付帳款零用金-v0.2.md` §1）
- 加盟店**各自收款 / 各自月結**
- 總部**只代**總倉供應商付款

## 應付帳款 Q1~Q6 的決定（2026-04-23 裁定，已套用此立場）

| Q | 問題 | 決定 | 如何呼應 C 混合型 |
|---|---|---|---|
| Q1 | 發票比對 | 手工輸入 `supplier_invoice_no`，P1 朝財政部電子發票平台拉取 | 加盟店自主輸入（不集中 OCR 服務）|
| Q2 | 預算管控 | 警示不擋 + **per-store 自設**上限；未設 = 不啟用 | 加盟店自主、總部看得到彙總 |
| Q3 | 跨店代墊 | 不發生（員工不跨店）→ 現有 `store_id` 自填 | 不適用 |
| Q4 | 陸貨外匯 | 下單日匯率鎖 TWD，不做多幣別 | v1 簡化（與立場無關）|
| Q5 | 加盟店當 supplier | **需要用才自動建**（on-demand） | 加盟店自主、不預先建立侵入資料 |
| Q6 | 加盟店間對付款 | **總部當 clearing house**：A → HQ → B | 總部看得到全部金流（可視性）+ 加盟主互不看帳戶（自主 / 隱私）|

## 如何套用到未來模組（操作指南）

### 🎯 設計功能時三問

1. **「這是誰的決定權？」**
   - 預設答：**加盟店**（除非列在「總部統一項」）
   - 例：促銷、預算、人事、通知樣式、取貨時段

2. **「總部能看什麼？」**
   - 預設答：**全部看得到**（彙總 + 明細都要）
   - admin role 在 RLS 裡有全租戶讀權限；一般 store_manager 只讀自己的 `store_id`

3. **「這是總部統一項嗎？」**
   - 檢查清單：供應鏈（總倉進貨）、品牌（logo / 店名格式）、會員（跨店通用）、ERP 系統、**發票廠商選定**
   - 不在清單上 = 加盟店自主

### 🚫 反模式

- 讓總部**強制**加盟店的促銷 / 價格 → 違反自主
- 加盟店**看得到其他加盟店**的敏感細節（銀行帳戶、員工薪資） → 違反自主 / 隱私
- 加盟店**自己決定**發票廠商 / 會員系統 → 違反「總部統一項」
- 總部**看不到**某家加盟店的營業額 → 違反可視性（除了加盟店自己的 LINE OA 訊息內容等隱私聊天紀錄）

## 權限設計預設（RLS）

```sql
-- 加盟店員工：只讀自己的 store
CREATE POLICY p_store_read_own ON <table>
  FOR SELECT USING (
    store_id = current_setting('app.store_id')::BIGINT
    OR current_user_role() = 'hq_admin'
  );

-- 總部 admin：全讀
CREATE POLICY p_hq_read_all ON <table>
  FOR SELECT USING (current_user_role() = 'hq_admin');

-- 加盟店員工：只寫自己的
CREATE POLICY p_store_write_own ON <table>
  FOR INSERT WITH CHECK (store_id = current_setting('app.store_id')::BIGINT);
```

每個新表都要走這套「store-owned + hq-admin all-read」模式。

## 這份 ADR 的例外 / 待議

- **v2 可能調整**：若總部真的需要干預某些營運（例如統一促銷檔期），這份 ADR 需要升版
- **如果加盟店不想讓總部看**：違反 ADR，v1 不支援；討論時請重新開 ADR
- **跨模組衝突**：若有兩個 PRD 對同一問題給不同答案，以這份 ADR 為準
